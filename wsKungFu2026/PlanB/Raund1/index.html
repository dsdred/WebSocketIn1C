WebSocket Echo Demo

Клиент для проверки эхо-сервера с авторизацией. Сервер поддерживает точную проверку `x-*` headers для Bruno и Node-клиентов, а этот браузерный demo использует стандартный subprotocol как совместимый transport для тех же учётных данных.

Браузерный API `WebSocket` не позволяет отправлять произвольные HTTP-заголовки вроде `x-login` или `x-token`. Поэтому этот HTML-клиент не может буквально выполнить `new WebSocket(url, { headers })`. Для ручного UI-теста здесь используется `Sec-WebSocket-Protocol`, а точную header-based авторизацию из задания проверяйте через Bruno или Node-клиент с библиотекой `ws`.
