========================
CONFIG.JSON
========================

Создай файл config.json.

Вынеси туда настройки:

{
  "port": 8080,
  "inactiveTimeoutMs": 30000,
  "pingIntervalMs": 10000,
  "targetScore": 100
}

Где:
- port — порт WebSocket-сервера
- inactiveTimeoutMs — таймаут завершения неактивного сеанса
- pingIntervalMs — интервал ping-pong проверки соединений
- targetScore — финальное число для победы, по умолчанию 100

Сервер должен читать настройки из config.json.

========================
ИГРОВАЯ ЛОГИКА
========================

Сервер хранит общее число:

score

Начальное значение:

score = 0

При подключении каждого клиента сервер случайно назначает ему команду:

- plus
- minus

Если команда plus:
- клиент видит кнопку "+"
- при клике отправляет действие plus
- сервер увеличивает общий score на 1

Если команда minus:
- клиент видит кнопку "-"
- при клике отправляет действие minus
- сервер уменьшает общий score на 1

Общий score должен обновляться онлайн у всех подключенных клиентов через WebSocket.

Победа:

- если score >= targetScore, побеждает команда plus
- если score <= -targetScore, побеждает команда minus

targetScore брать из config.json.

При достижении победы:
- сервер рассылает всем клиентам событие victory
- в сообщении указать победившую команду
- клиент показывает поздравление текстом на форме
- score сбрасывается в 0
- новое значение score рассылается всем клиентам

После первого следующего клика игрока поздравление должно скрываться.

========================
JSON-ПРОТОКОЛ
========================

Все сообщения между клиентом и сервером должны быть в JSON.

При подключении сервер отправляет:

{
  "type": "welcome",
  "team": "plus",
  "score": 0,
  "targetScore": 100
}

После изменения score сервер рассылает:

{
  "type": "state",
  "score": 1,
  "targetScore": 100
}

Клиент отправляет действие:

{
  "type": "action",
  "action": "plus"
}

или:

{
  "type": "action",
  "action": "minus"
}

При победе сервер отправляет:

{
  "type": "victory",
  "winner": "plus",
  "message": "Команда plus победила!",
  "score": 100,
  "targetScore": 100
}

После этого сервер должен сбросить score в 0 и разослать новое состояние:

{
  "type": "state",
  "score": 0,
  "targetScore": 100
}

При ошибке сервер отправляет:

{
  "type": "error",
  "message": "Описание ошибки"
}

========================
PING-PONG
========================

Обязательно реализуй ping-pong.

Требования:
- сервер регулярно отправляет ping всем клиентам
- клиент должен отвечать pong автоматически средствами WebSocket
- используй механизм ws.ping() / pong event на сервере
- храни флаг isAlive для каждого соединения
- при получении pong устанавливай isAlive = true
- если клиент не отвечает на ping, заверши соединение через terminate()

Интервал ping бери из config.json:

pingIntervalMs

========================
ТАЙМАУТ НЕАКТИВНОСТИ
========================

Обязательно реализуй завершение неактивных сеансов по таймауту.

Требования:
- время таймаута бери из config.json:
  inactiveTimeoutMs
- для каждого клиента храни lastActivityAt
- обновляй lastActivityAt:
  - при подключении
  - при получении любого валидного сообщения от клиента
  - при pong
- периодически проверяй неактивных клиентов
- если клиент неактивен дольше inactiveTimeoutMs, закрывай соединение:
  ws.close(4000, "Inactive timeout")
  или ws.terminate()

Логируй отключение по таймауту.

========================
CLIENT: PUBLIC/INDEX.HTML
========================

Создай простой клиент:

public/index.html

Файл должен работать как локальный HTML-файл без HTTP-сервера.

Клиент должен подключаться к:

ws://localhost:8080

Так как браузер не может напрямую читать config.json при открытии file://, в клиенте можно указать:

const WS_URL = "ws://localhost:8080";

Интерфейс клиента:

- статус подключения
- отображение команды пользователя:
  plus или minus
- отображение общего score
- отображение targetScore
- одна игровая кнопка:
  "+" для команды plus
  "-" для команды minus
- блок поздравления/победы
- лог событий

Поведение:
- при открытии страницы клиент может подключаться автоматически
  или через кнопку "Подключиться"
- после welcome показать команду пользователя
- если team = plus, показывать кнопку "+"
- если team = minus, показывать кнопку "-"
- при клике отправлять action в JSON
- общий score обновлять при получении state
- при victory показывать поздравление
- после первого следующего клика скрывать поздравление
- при disconnect показывать статус disconnected
- при ошибке показывать ошибку в логе

========================
BRUNO COLLECTION
========================

Подготовь Bruno WebSocket collection.

Важно:
- использовать именно WebSocket requests
- НЕ создавать HTTP/REST requests
- каждый .bru файл должен иметь:

meta {
  type: ws
}

Использовать:

ws://localhost:8080

Не использовать environment variables без необходимости.

========================
BRUNO REQUESTS
========================

1. ws-connect-player-a.bru

Описание:
Подключение игрока A.

Должен просто открывать WebSocket-сессию.

2. ws-connect-player-b.bru

Описание:
Подключение игрока B.

Должен просто открывать WebSocket-сессию.

3. ws-plus-action.bru

Описание:
Отправка действия plus.

Body:

{
  "type": "action",
  "action": "plus"
}

4. ws-minus-action.bru

Описание:
Отправка действия minus.

Body:

{
  "type": "action",
  "action": "minus"
}

5. ws-invalid-message.bru

Описание:
Отправка невалидного JSON или неизвестного action для проверки ошибок сервера.

Пример:

{
  "type": "unknown"
}

========================
ПРИМЕР BRUNO .BRU
========================

meta {
  name: ws-plus-action
  type: ws
  seq: 3
}

ws {
  url: ws://localhost:8080
}

body:ws {
  name: message 1
  content: '''
{
  "type": "action",
  "action": "plus"
}
'''
}
