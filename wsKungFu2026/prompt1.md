========================
АУТЕНТИФИКАЦИЯ
========================

Используй аутентификацию WebSocket через HTTP headers.

НЕ используй:
• query parameters
• ?login=...
• ?token=...

Используй только headers.

Поддержи 2 варианта аутентификации:

1. Login/password через headers:
   x-login: admin
   x-password: 123

2. Token через header:
   x-token: secret123

Захардкодь учётные данные:
• login: admin
• password: 123
• token: secret123

Проверяй headers при каждом новом подключении через:
req.headers

Логика авторизации:
• Если переданы x-login и x-password:
  - сравнивай с admin / 123

• Если передан x-token:
  - сравнивай с secret123

• Если ни один вариант auth невалиден:
  - закрывай соединение:
    ws.close(4001, 'Unauthorized')
    или
    ws.terminate()

• Если auth успешен:
  - отправляй клиенту JSON:
    {
      "status":"connected",
      "message":"WebSocket connected"
    }

• Добавь подробные комментарии к логике проверки headers.

========================
ТРЕБОВАНИЯ К BRUNO
========================

Создай Bruno-коллекцию в папке:
bruno/

Важно:
• Коллекция должна быть именно для WebSocket.
• НЕ создавай HTTP/REST-запросы.
• Каждый файл запроса Bruno должен иметь:

meta {
  type: ws
}

Структура:
- bruno/bruno.json
- bruno/collection.bru
- bruno/ws-login-pass.bru
- bruno/ws-token.bru
- bruno/ws-invalid-auth.bru

Файл bruno/bruno.json:

{
  "version": "1",
  "name": "websocket-echo-demo",
  "type": "collection"
}

Каждый WebSocket-запрос должен использовать Bruno WebSocket syntax:

meta {
  name: ws-login-pass
  type: ws
  seq: 1
}

ws {
  url: ws://localhost:8080
}

headers {
  x-login: admin
  x-password: 123
}

body:ws {
  name: message 1
  content: '''
Hello from Bruno
'''
}

Создай 3 WebSocket-запроса:

1. ws-login-pass.bru
headers:
• x-login: admin
• x-password: 123

2. ws-token.bru
headers:
• x-token: secret123

3. ws-invalid-auth.bru
headers:
• x-login: wrong
• x-password: wrong

Для каждого запроса:
• обязательно используй:
  meta {
    type: ws
  }

• обязательно используй:
  ws {
    url: ...
  }

• обязательно добавь:
  body:ws

• НЕ используй:
  - HTTP requests
  - GET/POST
  - REST syntax
  - query params
