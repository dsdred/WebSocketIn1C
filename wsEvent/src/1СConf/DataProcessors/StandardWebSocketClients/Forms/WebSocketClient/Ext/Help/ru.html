Настройка WebSocket-клиента

У объекта WebSocket-клиент для настройки доступны следующие параметры:
  • Метаданные - объект метаданных, на основании которого будет создан данный WebSocket-клиент,
  • Ключ - идентификатор WebSocket-клиента. Ключ должен быть уникальным и не может быть пустой строкой.
При изменении существующего клиента метаданные и ключ менять нельзя.
  • Предопределенный - если указан, такой WebSocket-клиент нельзя явно создать и удалять из встроенного языка. Он всегда существует в единственном экземпляре, которые создается в информационной базе при обновлении конфигурации после того, как в 
Нельзя сделать предопределенного клиента непредопределенным, и наоборот (флажок Предопределенный недоступен для изменения).

  • Подключать автоматически - указывает, что подключение к серверу надо выполнять автоматически при запуске системы "1С:Предприятие" и переустанавливать соединение при разрыве связи.
  • URL сервера - адрес WebSocket-сервера, с которым будет осуществляться работа с данным WebSocket-клиентом. Должен быть задан, если он не устанавливается в обработчике Перед подключением в модуле метаданных. URL-адрес должен начинаться со схемы ws:// или wss://.

Параметры соединения

  • Пользователь -  имя пользователя, который используется для стандартной аутентификации или аутентификации ОС на WebSocket-сервере
  • Пароль - пароль пользователя, который используется для стандартной аутентификации или аутентификации ОС на WebSocket-сервере..
  • Использовать аутентификацию ОС - 
  • Использовать прокси ОС - если  флажок установлен, то при установке соединения с WebSocket-сервером будут использованы настройки прокси-сервера, указанные в настройках операционной системы того компьютера, с которого выполняется соединение. 
  • Таймаут -  таймаут установки соединения с WebSocket-сервером (в секундах).

В списке Заголовки можно добавить дополнительные заголовки в HTTP-запрос GET (c заголовком Upgrade: websocket) на установку соединения. Таким образом, например, можно указать токен для авторизации.
